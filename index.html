Step-by-step

Prerequisites

- Up to 12 3D printers
– A Github account (host your site)
– A new Google account (send emails)
– A cheap PC  with a bunch of usb ports (connects to all the printers Note: this pc will be a server for the farm)
– A main PC (can be any computer Note: this pc will not be limited by the farm but will need to always be on)
-General knowledge of Javascript and HTML, you'll likely want to edit your website.

Step-1 (setting up your printers)

Before you begin, your printers must be able to print consistently had have a auto bed clearing script​. We used a combination of Creality Ender 3,v2 and Sovol SV06 with glass and PEI build plates.

Bed clearing
Idealy, you would use these Creality build plates as they release your object at about 37C.
Although HERE is a great solution for flexible buildplate.

A basic be clearing script for both flexible and Creality build plates can be found HERE under /BED_CLEARING_SCRIPT/bed_clearing.gcode


Filament runout (optional)
Investing a few buck in a few filament runout sensors will save you a lot of head aches.
Here is a cheap runout sensor from aliexpress that worked for me

Step-2 (set up your octoprint server)

There are many ways to run a server for all your printers. We went with one computer running all of our ​octoprint instances. 
Heres how:
-Install Ubuntu desktop on your server PC. (I recommend using 'pi' as a username for compatibility with the code)
-Once you've installed ubuntu, run:
sudo apt-get update  and  sudo apt-get upgrade 
-Reboot.
-Run:
sudo apt-get install git-all
-Lets download octoprint deploy using :
git clone https://github.com/paukstelis/octoprint_deploy
-Next launch it using:
sudo octoprint_deploy/octoprint_deploy.sh
-Select :
Prepare
-You will be prompted for udev detection, select:

Yes

-To add printers, select:
add instance

-Follow along with what it says from now on. I recommend naming your printer 1,2,3,... so you can essentially copy paste my work.
-If you need assistance with this step, refer to this as a source.

Step-3 (set up a samba server)

To send files to Octoprint, we will use an arrangement of network drives. 
First, run:
apt-get install -y net-tools
Next, install samba:
sudo apt install samba
Next, add your user to samba:
sudo smbpasswd -a "your username"  (ideally your username is 'pi')
You will be prompted to type a new password (password doesn't matter)

Next, we need to take ownership of the files we want to communicate with as the samba user account. Since we have multiple folders to take (1 for each printer) it's easier to take the entire directory they are stored in. To do so, run:
cd.. until you get to /








Next run:
sudo chown 'your username' home -R and sudo chgrp 'your username' home -R
Next we need to edit the samba config file. Do:
sudo nano /etc/samba/smb.conf
Now that we are viewing the file, delete all of the contents in the file.
Paste this at the top of the file:
[global]
force create mode = 0664
server string = File Server
workgroup = LLTV
security = user
map to guest = Bad User
name resolve order = bcast host

.. one of these for each printer:
[1]
path = /home/pi/.1/watched
force create mode = 0664
create mask = 0664
force user = pi
force group = pi
directory mode = 0775
public = yes
writable = yes

You will need to paste one of those for each printer. Change the '1' in [ 1​​​ ​​] and path = /home/pi/.1/watched to the number correlating to your printers.
If you named your octoprint instances something other than 1,2,3... or have a different username than 'pi'. If thats the case simply change:
path = /home/[your username]/.[instance name]/watched. 

HERE under /SambaServerData/smb_conf.txt is a setup for printers 1-12 and a username 'pi'

This is the directory we need to take ownership  from. If you do "ls -l" you will see 'root' currently owns it​.

This is the directory we begin in

These titles need to be green

Step-5 (set up the Octoprint instances)

We now need to set it so octoprint starts a print once it receives a file and notifies our system. 
To make is octoprint starts a print on file received, we will use a modified version of the Autoselect pluggin found HERE under /Octoprint_pluggins/autoselct.zip
For the notification we will use the Webhooks pluggin found Here under /Octoprint_pluggins/Webhook.zip

To add these to your instances, go to your octoprint instances by searching
 'your-servers-ip-address':500#
where #=0,1,2,3... in order of which instance you made first.
Ex: if you made instance 1 first, you search 'your-servers-ip-address':5001 to find it

On each instance add both pluggins by going to settings->pluggin manager->get more than add both files with 'from an uploaded file'















We now need to configure the Webhook pluggin. Navigate to settings->Webhooks->Webhook parameters->and change device identifier to the number of your instance.

















We are going to come back to this step on step 8... Once you have completed step 8, take the URL form the webhook pluggin and paste in 'webhook ur'l.
REMEMBER TO DO THIS STEP FOR ALL INSTANCES!

Step-6 (set up the Main PC's code)

Setting up the PC's code is as easy as it can get. We simply need to copy all the files of THIS drive to the root of your new Google accounts drive, then link your drive to your main PC to a new file located in C:\drive.



Let's first start by making that file. Navigate to C:\ on your main PC. In C:\ make a new folder called 'drive'
Next go HERE to download drive desktop.
Once it's downloaded, follow the following to open it

















settings>preferences















Under "My Drive syncing options," select Mirror files. As a folder to mirror to, select the 'drive' folder we made.

Now go HERE and copy all the file to c:\drive

Step-7 (set up a Google spreadsheet)

In this step, we will create and format a Google spreadsheet to our liking.
-Firstly, create a new google spreadsheet. Give it a name.
-Next create a new sheet in the spreadsheet, keep the names of the sheets sheet1 and sheet2 respectivaly.
-In sheet1 in the headers, label them in this order: A(Name), B(Email), C(Colour),D(File),E(file.txt)
-In sheet2 in the headers, label them in this order:A(timestamp_incoming_webhook), B(topic), C(deviceIdentifier), D(extra), M(Name.gcode), N(state of printer), O(Name.stl), P(row in test), Q(time(normal)
-It's better to copy/paste the contents of the brackets to their respective header to insure it works.  

Step-8 (get webhook pluggin)

To get status updates on your printer, we will get webhooks and put them in your spreadsheet. To do this, we will use a plugin called webhook for sheets. 
In your spreadsheet, navigate to the extentions->add-ons->get add-on->search "webhooks for sheets"
-Once you find it, install it.









-To proceed, find webhook for sheets on the right side
menu. 
-Continue by doing what the plugin says until you get 
your webhook link. Return to step-5 and paste the link.

Step-9 (set up the website)

For simplicity sake, we will create a web app using apps scripts.


-Start on your spreadsheet, navigate to your extentions->apps scripts

-Name your new project.

-Go HERE under /apps-scripts/code_gs.txt and paste the contents to your code.gs in apps scripts.

-Now in apps scripts, create a new html by clicking the + ->html. Name if 'form'.

-Go HERE under /apps-scripts/form_html.txt and paste it's contents to form.html in apps scripts.



Now deploy the web app




















The link given after deployment is the link to your site. Test your form and make sure data gets filled into the sheet and file in '/3D printing request/please upload your file here'.

Step-10 (set up webhook receiver)

We hav​e now set up the form to receive files, the PC will now open files in the slicer once the file is received, once you save your sliced file, it will queue and send it to an available printer via octoprint, octoprint returns a webhook on printer start and whens it's done. 
Now we need to take these webhooks and notify the system the printer is currently printing and cannot receive anymore file until it receives a 'done' webhook.

-Start on your spreadsheet, extentions->apps-script and create a new project again.

-Name the project 'webhook Receive'

-Now go HERE and copy the contents under /apps-scripts/webhook-received_gs.txt

-Paste in your new project in code.gs.

-Now on the left menu, navigate to triggers->add trigger. Under 'Choose which function to run' select 'getHook' , under 'select event source' select 'time-driven' under 'Select type of time based trigger' select 'minute timers' and under 'Select minute interval' select 'every minute'-> SAVE

Simple!

Things to note before testing

-Firstly, your main PC must be set to open STL files with your slicer of choice by default.

Configuring colour
Your printer may have up to 4 different colours, you can pick which octoprint instance receives what colour with the config.txt file in the google drive. You may also disable printers there and plug in your servers IP addresse

Note: octoprint instances may change printers after restarting. if you have many colours, sending your file to octoprint instance 1 doesn't mean it's being sent to printer 1. After every restart, it is recommended to many sure each printer is connected to it's respected instance. 



On farm startup or restart or power outage or whatever
When you wish to start the farm, there are 2 components that you must make sure are running. Firstly, your server must be running, connected to the internet with a static IP. 
Secondly, your main PC must be running it's powershell scripts. There are 2 scripts, one that will detect oncoming files on form received and another to detect sliced files and queue and send to printers. 
To start them simply open two windows powershell tabs, click HERE and paste the contents of '/DATA/gcode.ps1' in one tab and the contents of /DATA/stl.ps1 in the other.
